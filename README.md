## 💁‍♂️ 스터디 방법

### 1️⃣ 커밋 규칙

| Type | 내용 |
|--|--|
| solved | 해결함 |
| fix | 수정 |
| refactor | 리펙토링 |
| docs | 문서관련 |

```
< TYPE > : < SUB >
< BLANK >
< DETAIL > 
```
```
❓(예시)
solved : P_피보나치수열_1103 해결

재귀함수 방식으로 접근하여 해결
```

### 2️⃣ 디렉토리 구조

* N : 해당일 수
* `본인별명`.`weekN`.`dayN` 순으로 디렉토리를 구성한다.

```
❓(예시)
koo.week1.day1
```

### 3️⃣ 파일명

* `P` _ `문제이름` _	`문제번호`  순으로 작성한다.
* 문제이름은 공백없이 붙여서 작성한다.
 ```
❓(예시) 
P_피보나치함수_1003.java
```

### 4️⃣ 깃 브랜치 구조

| 브랜치 명 | 포함될 내용 | 비고 |
|:----------|:----------:|----------:|
| master | 메인 브랜치 | main branch |
| solved | 해결완료 | sub branch |
| 각자 별명 | 개인 브랜치 | private | 

```
❓(예시) 
koo 브랜치 생성
```

### 5️⃣ 참여 방법
1. 깃 레포지토리 `fork`
2. `새로운 브랜치` 생성 ( 본인 별명 )
3. 원격저장소의 브랜치 생성확인
4. pull 하여 `로컬저장소`에서 관리
5. 풀이 한 소스코드를 `commit / push`
6. `PR` 실시
7. 코드리뷰
